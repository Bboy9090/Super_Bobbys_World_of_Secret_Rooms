# Contributing to REFORGE OS

Thank you for your interest in contributing to REFORGE OS!

---

## What We Accept

✅ **UI/UX improvements**
✅ **Documentation enhancements**
✅ **Testing additions**
✅ **Compliance logic improvements**
✅ **Reporting enhancements**

---

## What We Do NOT Accept

❌ **Exploit code**
❌ **Bypass workflows**
❌ **Jailbreak/root instructions**
❌ **Automation of circumvention**
❌ **References to internal Pandora Codex material**

**Violations will be rejected immediately.**

---

## Contribution Rules

1. **All changes must preserve auditability**
2. **All features must respect ownership and jurisdiction**
3. **Language must remain compliance-safe**
4. **No execution logic — only analysis and routing**

---

## Development Setup

```bash
git clone https://github.com/YOUR-ORG/REFORGE-OS.git
cd REFORGE-OS
# Follow platform-specific setup instructions
```

---

## Code Review Process

1. Create a feature branch
2. Make your changes
3. Ensure compliance checks pass (see `.github/workflows/compliance-guard.yml`)
4. Submit a pull request
5. Wait for review and approval

---

## Compliance Checks

All contributions must pass:
- Language scan (no forbidden terms)
- Pandora Codex isolation check
- Auditability verification
- Legal posture validation
